# 구글 OAuth 설정 가이드

구글 클라우드 콘솔에서 `GOOGLE_CLIENT_ID`와 `Google Client Secret`을 발급받는 방법입니다.

## 1. 프로젝트 생성
1. [Google Cloud Console](https://console.cloud.google.com/)에 접속합니다.
2. 좌측 상단의 프로젝트 선택 드롭다운을 클릭하고 **새 프로젝트**를 선택합니다.
3. 프로젝트 이름(예: `OAuth System`)을 입력하고 **만들기**를 클릭합니다.

## 2. OAuth 동의 화면 구성
1. 좌측 메뉴에서 **API 및 서비스** > **OAuth 동의 화면**으로 이동합니다.
2. **외부(External)**를 선택하고 **만들기**를 클릭합니다.
3. 필수 정보를 입력합니다:
   - **앱 이름**: OAuth System
   - **사용자 지원 이메일**: 본인 이메일 선택
   - **개발자 연락처 정보**: 본인 이메일 입력
4. **저장 후 계속**을 클릭하여 다음 단계로 넘어가고, 대시보드로 돌아옵니다 (범위 설정 등은 건너뛰어도 됩니다).

## 3. 사용자 인증 정보(Credentials) 생성
1. 좌측 메뉴에서 **API 및 서비스** > **사용자 인증 정보**로 이동합니다.
2. 상단의 **+ 사용자 인증 정보 만들기** > **OAuth 클라이언트 ID**를 선택합니다.
3. **애플리케이션 유형**으로 **웹 애플리케이션**을 선택합니다.
4. 이름은 `Web Client` 등으로 지정합니다.
5. **승인된 리디렉션 URI** 항목에 다음 주소를 추가합니다:
   - 로컬 테스트용: `http://localhost:3000` (메인 페이지 주소)
   - *주의: 포트폴리오 사이트 주소가 있다면 그 주소도 추가해야 합니다. (예: `https://my-site.vercel.app/`)*
6. **만들기**를 클릭합니다.

## 4. 키 확인 및 사용
1. 생성 완료 팝업에서 **클라이언트 ID**와 **클라이언트 보안 비밀번호(Secret)**를 복사합니다.
2. 이 키들을 복사하여 Playground 페이지에 입력하거나, `.env` 파일에 설정하여 사용합니다.

## 5. (선택사항) People API 활성화
1. **API 및 서비스** > **라이브러리**로 이동합니다.
2. `Google People API`를 검색하고 **사용**을 클릭합니다. (더 정확한 프로필 정보를 가져오기 위함)
